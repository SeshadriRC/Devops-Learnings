> **An Auto Scaling Group automatically maintains the desired number of EC2 instances by launching or terminating instances based on demand or health.**

**Eg:** If in case 2 servers able to handle only 200 requests, but servers are receving 300 requests then automatically it will spin up additional EC2 instances

### Key points:

* Ensures **high availability**
* Automatically **scales out** (add instances)
* Automatically **scales in** (remove instances)
* Replaces **unhealthy instances**
* Uses **Launch Template / Launch Configuration**

### Common use case:

* Works with **Load Balancer**
* Handles traffic spikes automatically
